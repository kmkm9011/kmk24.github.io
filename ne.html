문제점

구분이 안 되는 서비스

    NexR Enterprise는 11개의 서비스를 제공하기로 기획되었으며, 이 때문에 흑백 컬러의 기획서가 나왔을 때 서비스가 구분이 안 된다는 의견이 많았습니다. 사용자가 서비스를 헷갈리는 경우, 오 클릭과 이탈을 유발할 수 있습니다. 따라서 문제점 파악 및 “서비스 구분율 증대"와 “이탈률 감소"를 목표로 프로젝트를 진행했습니다.

    다른 형태를 사용할 수 없음으로, 같은 컬러를 사용할 시 서비스들이 헷갈릴 수 있어 사용할 수 없습니다.

문제 01 - 로고 결과

개성과 통일성

# 로고 디자인

다른 컬러 / 비슷한 형태로 로고를 디자인하자
개성 - 다른 컬러

그라데이션

NexR Enterprise의 제품은 성격상 Table(표) 형식이 많으므로, 한눈에 다른 서비스임을 구분할 수 있도록 컬러에서 개성을 표현하고자 했습니다. 하지만 11개의 서비스가 컬러가 뚜렷이 구분되어야 하므로, 사용할 수 있는 컬러가 제한적이라는 문제가 발생했습니다. 이를 해결하기 위해 그라디언트 컬러를 사용하여 컬러 사용의 폭을 넓히고, dentity 컬러로 각 서비스를 구분할 수 있도록 로고를 제작했습니다.

통일성 - 비슷한 형태

형태의 규칙성

빅데이터를 상징할 수 있는 'Data Lake'를 선택했습니다. 여기서 한 번 더 단순화를 거쳐 '물결' 아이콘을 빅데이터의 상징으로 삼았으며, 동시에 로고의 통일성을 주기 위한 장치로 사용하게 되었습니다.또한 로고에 알파벳을 숨겨 개성과 재미를 더하고, 각 서비스를 상징할 수 있는 메타포를 사용하여 서비스의 Identity를 챙기면서 규칙성을 생성하여 동시에 통일감을 줄 수 있도록 의도했습니다.

# UI 디자인

다른 컬러 / 비슷한 형태로 UI를 디자인하자
개성 - 다른 컬러

그라데이션

로고와 동일하게 그라데이션 컬러를 활용하여 UI를 구성했습니다. 특히 넓은 영역을 차지하는 Header나 Menu 영역에 그라데이션을 적극적으로 활용하여 한눈에 어떤 서비스를 사용 중인지 사용자가 인지할 수 있도록 설계했습니다.

통일성 - 비슷한 형태

디자인 시스템

UI에 통일성을 부여하고 여러 서비스에 적용할 수 있도록 디자인 시스템을 제작했으며, 효율적인 관리를 위해 포토샵에서 피그마로 전환했습니다. 그 결과, 프론트 개발 코드 길이를 단축하여 개발 효율성 증대되었고, 관리 비용을 대폭 절감하여 작업 시간이 3개월 이상 단축되었습니다.
